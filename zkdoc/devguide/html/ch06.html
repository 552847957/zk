Component Sets and XML Namespaces
Standard Namespaces

The ZK User Interface Markup Language (ZUML) is based on XML. Each XML element describes what component to create. A XML attribute describes an initial values to be assigned to the created component. An XML processing instruction describes how to process the whole page, such as the page title.

Different sets of components are distinguished by XML namespaces. For example, the namespace of XUL is http://www.zkoss.org/2005/zul,[32] and that of XHTML is http://www.w3.org/1999/xhtml.


[32] It was called http://www.mozilla.org/keymaster/gatekeeper/there.is.only.xul. However , many non-XUL features are added, so it is better to use an independent namespace.
